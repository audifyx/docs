# WHY THE ECOSYSTEM EXISTS

Current crypto workflows are heavily fragmented.

Users currently split activity across:
- X
- Telegram
- Discord
- Dexscreener
- Solscan
- Birdeye
- wallet trackers
- analytics tools
- launch bots
- trading terminals

This creates major inefficiencies.

Important information becomes fragmented across multiple systems.

Communities become disconnected.

Realtime coordination becomes difficult.

Launch discovery becomes noisy.

Fake deployments flood ecosystems.

Users struggle to identify trustworthy launches.

Users struggle to determine which launch is the real canonical origin.

SolTools + OGSCAN were created to solve these fragmentation problems.

# OGSCAN

# WHAT IS OGSCAN?

OGSCAN is a launch-intelligence and token-lineage reconstruction system focused on Solana ecosystems.

Traditional token scanners usually index:
- the first visible pair
- the first indexed contract
- the first liquidity event discovered by the scanner

OGSCAN instead attempts to determine:
- the actual launch lineage
- migration continuity
- canonical token origin
- deployer behavior patterns
- wallet ecosystem relationships
- launch-state behavior

This creates a fundamentally different intelligence model.

---

# THE CORE PROBLEM OGSCAN SOLVES

Solana launches are extremely fragmented.

Common ecosystem problems include:
- duplicated ticker deployments
- migration confusion
- cloned contracts
- fake OG tokens
- spam launch flooding
- bundled launches
- liquidity migration
- relaunch cycles
- fake community launches

Traditional scanners often fail because they treat launches as isolated contracts.

OGSCAN instead interprets launches as connected behavioral ecosystems.

# CORE OGSCAN SYSTEMS

## 1. TOKEN LINEAGE RECONSTRUCTION

The system attempts to reconstruct historical relationships between:
- migrated tokens
- relaunched ecosystems
- deployer networks
- liquidity movement
- wallet continuity

Purpose:
Determine whether different contracts actually belong to the same ecosystem lineage.

---

## 2. LAUNCH-STATE INTERPRETATION

The system analyzes:
- liquidity sequencing
- migration timing
- wallet participation
- holder distribution
- deployer behavior
- ecosystem continuity

Purpose:
Interpret launch behavior instead of only indexing visible contracts.

---

## 3. WALLET GRAPH ANALYSIS

OGSCAN attempts to cluster relationships between:
- deployers
- bundled wallets
- liquidity providers
- migration wallets
- repeat launch actors
- suspicious wallet networks

Purpose:
Identify behavioral ecosystem relationships.

---

## 4. MIGRATION CONTINUITY

The system evaluates:
- migration sequencing
- liquidity continuity
- deployer continuity
- wallet continuity
- community continuity

Purpose:
Determine whether a later token is the actual continuation of an earlier ecosystem.

---

## 5. LIQUIDITY SEQUENCING

The system evaluates:
- LP creation timing
- liquidity acceleration
- migration flow
- liquidity abandonment
- coordinated launch patterns

Purpose:
Separate legitimate launches from spam or manipulated deployments.

---

## 6. BUNDLE DETECTION

The system analyzes:
- concentrated supply
- coordinated wallets
- launch concentration
- suspicious holder structures
- bundled launch behavior

Purpose:
Evaluate ecosystem trust signals.
